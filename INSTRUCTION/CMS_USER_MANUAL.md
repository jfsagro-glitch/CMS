## Введение

CMS (Collateral Management System) - это веб-система для управления залоговым имуществом банков и финансовых организаций. Система позволяет:

- Ведение реестра залогового имущества
- Управление залоговым портфелем
- Мониторинг и переоценка обеспечения
- Формирование аналитических отчетов
- Управление задачами и документами

### Системные требования

- **Браузер:** Chrome 90+, Firefox 88+, Edge 90+, Safari 14+
- **Интернет:** Требуется для первоначальной загрузки
- **Разрешение экрана:** Рекомендуется 1920x1080 или выше

### Первый запуск

1. Откройте браузер и перейдите по адресу: **https://cmsauto.ru/**
2. При первом запуске система автоматически загрузит демо-данные
3. Дождитесь завершения загрузки (индикатор "Загрузка..." исчезнет)

## Основные модули

### Реестр

**Назначение:** Управление карточками залогового имущества

**Путь:** Главное меню → Реестр

#### Основные возможности:

1. **Просмотр карточек**
   - Таблица со всеми карточками залогового имущества
   - Сортировка по столбцам
   - Фильтрация по категориям, статусам, кодам ЦБ

2. **Создание новой карточки**
   - Нажмите кнопку **"+ Создать карточку"**
   - Заполните форму с 5 вкладками:
     - **Основная информация:** номер, название, категория, классификация
     - **Партнеры:** залогодатели, заемщики, собственники
     - **Адрес:** местоположение объекта (с автозаполнением DaData)
     - **Характеристики:** динамические поля в зависимости от типа объекта
     - **Документы:** загрузка и управление документами

3. **Редактирование карточки**
   - Двойной клик по строке в таблице
   - Или кнопка "Редактировать" в меню действий

4. **Удаление карточки**
   - Кнопка "Удалить" в меню действий
   - Подтверждение удаления

5. **Экспорт данных**
   - Кнопка "Экспорт в Excel"
   - Экспорт всех карточек или отфильтрованных

> **Примечание:** Здесь должен быть скриншот страницы Реестра

#### Классификация объектов

Система поддерживает 3-уровневую классификацию:

- **Уровень 0:** Категория (Недвижимость, Движимое имущество, Имущественные права)
- **Уровень 1:** Вид объекта (Квартира, Офис, Оборудование и т.д.)
- **Уровень 2:** Тип (Здание, Помещение, Сооружение)

При выборе классификации автоматически определяется **код ЦБ** (код Центрального банка).

#### Характеристики объектов

В зависимости от выбранного типа объекта система автоматически показывает соответствующие поля:

- **Жилая недвижимость:** площадь, количество комнат, этаж, год постройки
- **Коммерческая недвижимость:** площадь, назначение, класс объекта
- **Оборудование:** модель, производитель, год выпуска, инвентарный номер
- И другие типы (всего 18 типов с характеристиками)

### Залоговый портфель

**Назначение:** Просмотр и анализ залогового портфеля

**Путь:** Главное меню → Залоговый портфель

#### Основные возможности:

1. **Поиск по портфелю**
   - Выберите атрибут для поиска:
     - ИНН
     - Залогодатель
     - Заемщик
     - REFERENCE залога
     - Номер договора залога
     - № договора
   - Введите значение в поле поиска

2. **Фильтрация**
   - По сегменту бизнеса (МБ, СРБ, КБ)
   - По группе
   - По ликвидности
   - По статусу мониторинга

3. **Статистика**
   - Общая сумма долга
   - Общая стоимость обеспечения
   - Покрытие долга
   - Количество сделок

4. **Таблица портфеля**
   - REFERENCE сделки
   - Информация о заемщике/залогодателе
   - Тип обеспечения
   - Сумма долга
   - Стоимость обеспечения
   - Покрытие долга (%)
   - Ликвидность
   - Статус мониторинга

5. **Действия со сделкой**
   - **Просмотр досье:** открытие детальной информации по сделке
   - **Создать задачу:** создание новой задачи по сделке
   - **Экспорт в Excel:** выгрузка данных портфеля

> **Примечание:** Здесь должен быть скриншот страницы Залоговый портфель

#### Просмотр досье сделки

При нажатии на кнопку "Просмотр досье" открывается модальное окно с:

- **Основная информация:** REFERENCE, заемщик, залогодатель, даты
- **Обеспечение:** список объектов залога с деталями
- **Документы:** список документов по сделке
- **История:** временная шкала событий по сделке
- **Задачи:** связанные задачи

### Мониторинг

**Назначение:** Планирование и контроль мониторинга и переоценки обеспечения

**Путь:** Главное меню → Мониторинг

#### Два подраздела:

##### 1. План мониторинга обеспечения

**Основные возможности:**

1. **Фильтрация по срокам**
   - Все
   - Просрочено
   - 7 дней
   - 30 дней
   - 90 дней

2. **Дополнительные фильтры**
   - По типу обеспечения
   - По виду/методу мониторинга
   - По ответственному

3. **Статистика**
   - Всего записей
   - Просрочено
   - В течение 7 дней
   - В течение 30 дней

4. **Таблица плана**
   - Сделка (REFERENCE, заемщик, сегмент)
   - Обеспечение (тип, ликвидность)
   - План мониторинга (дата, частота, статус)
   - Вид/метод мониторинга
   - Ответственный
   - Приоритет
   - Действия (открыть карточку мониторинга)

5. **Карточка мониторинга**
   - Детальная информация по мониторингу
   - История мониторингов
   - Планирование следующего

##### 2. План переоценок обеспечения

**Основные возможности:**

1. **Фильтрация** (аналогично плану мониторинга)

2. **Таблица плана переоценок**
   - Сделка
   - Обеспечение
   - План переоценки (дата, частота)
   - Стоимость (залог, рыночная)
   - Метод переоценки
   - Ответственный
   - Приоритет

3. **Карточка переоценки**
   - Детальная информация
   - История переоценок
   - Планирование следующей

> **Примечание:** Здесь должны быть скриншоты обоих подразделов Мониторинга

### Настройки

**Назначение:** Управление настройками системы

**Путь:** Главное меню → Настройки

#### Подразделы:

##### 1. Управление сотрудниками

**Путь:** Настройки → Управление сотрудниками

**Основные возможности:**

- Просмотр списка сотрудников
- Добавление нового сотрудника
- Редактирование данных сотрудника
- Управление ролями и правами доступа
- Деактивация сотрудника

**Поля сотрудника:**
- ФИО
- Должность
- Регион
- Роль (Руководитель, Менеджер, Оценщик и т.д.)
- Email
- Телефон
- Статус (Активен/Неактивен)

##### 2. Справочники

**Путь:** Настройки → Справочники

**Основные возможности:**

- Просмотр всех справочников системы
- Редактирование элементов справочников
- Добавление новых элементов
- Удаление элементов
- Активация/деактивация элементов

**Доступные справочники:**

1. **Регионы** - список регионов
2. **Сегменты бизнеса** - МБ, СРБ, КБ
3. **Группы** - группы портфеля
4. **Ликвидность** - уровни ликвидности
5. **Категории обеспечения** - категории из карточек сделок
6. **Типы мониторинга** - плановый, внеплановый, внеочередной
7. **Категории качества обеспечения** - I-V категории
8. **Атрибуты залогового имущества** - справочник атрибутов
9. **Атрибуты залога** - справочник из файла atr1.xlsx (928 атрибутов)

**Работа со справочниками:**

1. Выберите справочник из списка (вкладки)
2. Просмотрите элементы в таблице
3. Для добавления: кнопка "+ Добавить"
4. Для редактирования: кнопка "Редактировать" в строке
5. Для удаления: кнопка "Удалить" с подтверждением
6. Для активации/деактивации: переключатель в таблице

##### 3. Атрибуты залогового имущества

**Путь:** Настройки → Атрибуты залогового имущества

**Описание:** Прямой доступ к справочнику атрибутов залогового имущества

##### 4. Атрибуты залога

**Путь:** Настройки → Атрибуты залога

**Описание:** Прямой доступ к справочнику атрибутов залога (928 атрибутов из файла atr1.xlsx)

> **Примечание:** Здесь должны быть скриншоты всех подразделов Настроек

## Часто задаваемые вопросы

### Как создать новую карточку залогового имущества?

1. Перейдите в модуль **Реестр**
2. Нажмите кнопку **"+ Создать карточку"**
3. Заполните все вкладки формы
4. Нажмите **"Сохранить"**

### Как найти сделку в портфеле?

1. Перейдите в модуль **Залоговый портфель**
2. Выберите атрибут для поиска (ИНН, REFERENCE и т.д.)
3. Введите значение в поле поиска
4. Результаты отобразятся автоматически

### Как экспортировать данные?

В большинстве модулей есть кнопка **"Экспорт в Excel"** или **"Экспорт CSV"**:
- **Реестр:** Экспорт всех карточек
- **Портфель:** Экспорт портфеля
- **Задачи:** Экспорт CSV

### Как изменить тему интерфейса?

В правом верхнем углу есть переключатель темы (светлая/темная).

### Где хранятся данные?

Данные хранятся локально в браузере в IndexedDB. Это означает:
- Данные доступны офлайн
- Данные сохраняются между сессиями
- Данные привязаны к браузеру и домену

### Как добавить новый элемент в справочник?

1. Перейдите в **Настройки → Справочники**
2. Выберите нужный справочник
3. Нажмите **"+ Добавить"**
4. Заполните форму
5. Нажмите **"Сохранить"**

### Как работает автозаполнение адреса?

В форме создания карточки на вкладке **Адрес** используется сервис DaData:
1. Начните вводить адрес
2. Появится список подсказок
3. Выберите нужный адрес из списка
4. Поля заполнятся автоматически

### Что делать, если данные не загружаются?

1. Проверьте подключение к интернету
2. Обновите страницу (F5)
3. Очистите кэш браузера
4. Проверьте консоль браузера (F12) на наличие ошибок

## Техническая поддержка

При возникновении проблем:

1. Проверьте консоль браузера (F12 → Console)
2. Убедитесь, что используется поддерживаемый браузер
3. Попробуйте обновить страницу
4. Очистите кэш браузера
