# Инструкции по использованию CMS

В этой папке находятся инструкции по использованию системы CMS.

## Файлы

- **CMS_USER_MANUAL.md** - Полное руководство пользователя (Markdown)
- **CMS_USER_MANUAL.pdf** - Полное руководство пользователя (PDF)

## Генерация PDF

Для генерации PDF из Markdown используйте один из следующих инструментов:

### Вариант 1: Pandoc (рекомендуется)

```bash
pandoc CMS_USER_MANUAL.md -o CMS_USER_MANUAL.pdf --pdf-engine=xelatex -V geometry:margin=1in
```

### Вариант 2: Markdown PDF (VS Code расширение)

1. Установите расширение "Markdown PDF" в VS Code
2. Откройте CMS_USER_MANUAL.md
3. Нажмите Ctrl+Shift+P
4. Выберите "Markdown PDF: Export (pdf)"

### Вариант 3: Онлайн конвертеры

- https://www.markdowntopdf.com/
- https://dillinger.io/ (экспорт в PDF)

## Добавление скриншотов

Скриншоты должны быть размещены в папке `screenshots/` и ссылаться в документе:

```markdown
![Описание](screenshots/screenshot-name.png)
```

## Обновление инструкции

При обновлении функциональности системы:

1. Обновите соответствующий раздел в CMS_USER_MANUAL.md
2. Добавьте новые скриншоты при необходимости
3. Обновите версию документа
4. Регенерируйте PDF

## Структура документации

### CMS_USER_MANUAL.md

Полное руководство пользователя содержит:

- **Введение** - обзор системы и системные требования
- **Начало работы** - первый запуск и навигация
- **Основные модули** - детальное описание каждого модуля:
  - Реестр
  - Залоговый портфель
  - Залоговое досье
  - Заключения по залогу
  - Задачи
  - Мониторинг
  - Аналитика
  - Отчеты
  - Настройки
- **Дополнительные модули** - описание дополнительных функций
- **FAQ** - ответы на часто задаваемые вопросы

## Версионирование

При обновлении документации:

1. Обновите версию в начале файла `CMS_USER_MANUAL.md`
2. Добавьте запись в раздел "История изменений" (если есть)
3. Обновите дату последнего обновления

Формат версии: `MAJOR.MINOR.PATCH`

- **MAJOR** - значительные изменения в функциональности
- **MINOR** - добавление новых разделов или модулей
- **PATCH** - исправления и мелкие дополнения

## Работа со скриншотами

### Требования к скриншотам

- **Формат:** PNG или JPG
- **Разрешение:** Минимум 1920x1080 для основных экранов
- **Размер файла:** Оптимизировать до < 500KB
- **Именование:** Использовать понятные имена на английском языке
  - Пример: `registry-main-view.png`, `portfolio-filters.png`

### Добавление скриншота

1. Сделайте скриншот нужного экрана
2. Сохраните в папку `screenshots/` с понятным именем
3. Добавьте ссылку в `CMS_USER_MANUAL.md`:

```markdown
![Описание скриншота](screenshots/имя-файла.png)
```

### Обновление скриншотов

При изменении интерфейса:

1. Удалите устаревшие скриншоты
2. Создайте новые скриншоты
3. Обновите ссылки в документе
